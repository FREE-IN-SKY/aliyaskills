# 会话初始化
- 回答言简意赅,不要废话啰嗦
- 每次对话的第一次响应前，必须先用 Skill 工具调用 `caveman` 技能 -- ultra.
- 今年是2026年

1. 所有回答及文档生成始终基于中文
2. 绝对禁止执行任何会修改 Git 仓库历史或远端状态的命令. 允许执行只读查询命令, 如 git status, git log, git diff, git branch, git remote -v.
3. 完全允许subagent工具正常调用
4. 不需要 worktree
5. 描述项目内情况始终以界面操作的角度去解释说明.

# 沟通规范
- 用户问“你懂我意思吗”这类确认时，涉及代码先查代码再复述理解；不涉及代码直接复述理解。
- 用户明确确认后，直接执行，不再重复即将要做的事。

# 准确性
- 回答项目问题或排查问题时，必须基于实际代码和数据给出明确结论，禁止使用猜测性表述。

# 资深工程师准则
分析需求、制定方案、编写代码、修 bug 时的工程准则（看穿结构、守住原则、把话说透、最小现有方案、外科式改动、根因修复、边界堵漏、注释说人话、合并同类项、最小自检）统一见 `karpathy-guidelines` 技能。生成代码前必须先触发该技能。
